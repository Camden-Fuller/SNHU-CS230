# SNHU-CS230
Briefly summarize The Gaming Room client and their software requirements. Who was the client? What type of software did they want you to design?
The Client is The Gaming Room they currently have a Android game called Draw It or Lose It. They wanted to expand their Android game to a web based game that can run on multiple different operating platforms. The Software needs to support multiple teams, multiple players per team, unique names and idenfiers for each game, teams and players.

What did you do particularly well in developing this documentation?
I feel I did paticularly well with making sure the design decsisions were clear and concise.

What about the process of working through a design document did you find helpful when developing the code?
Working with a design document was helpful with developing the code because it forced me to think the softwares structure and constraints before implementation.

If you could choose one part of your work on these documents to revise, what would you pick? How would you improve it?
I would probably revise the evaluation section making sure to make the platform comparison more detailed and consistent. I would also include more specific examples of how each platform affects performance, security, scalability, and development cost.

How did you interpret the user’s needs and implement them into your software design? Why is it so important to consider the user’s needs when designing?
I focused on what The Gaming Room wanted the sofware to accomplish. I implemented these needs into the design by recommending a web-based, distributed system with centralized game-state management and secure client-server communication. Considering the user’s needs is important because software that does not match the client’s goals may function technically but still fail as a product.

How did you approach designing software? What techniques or strategies would you use in the future to analyze and design a similar software application?
I started with the requirements identifying the main objects in the system and then determining how they would interact. I used object-oriented design, inheritance, class relationships, and the singleton pattern to organize the application. In the future, I would use similar strategies, including requirement analysis, UML diagrams, platform evaluation, and early architectural planning, to design a similar software application more effectively.
